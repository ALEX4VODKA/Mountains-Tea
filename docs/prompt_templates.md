# Codex Prompt 模板

本文件保存常用的 Codex 任务提示词。

---

## 1. 建立目录结构

请为这个 Godot 4 项目建立基础目录结构。

要求：
1. 创建 scenes、scripts、assets、docs 等文件夹。
2. 目录结构要适合一个 2D 俯视角像素风休闲探索游戏。
3. 项目名为“Mountain Tea House / 山间茶铺”。
4. 只建立结构和必要的说明文档，不要写具体玩法代码。
5. 完成后告诉我：
   - 改了哪些文件
   - 每个目录的用途
   - 下一步推荐做什么

---

## 2. 玩家移动系统

请只实现 Godot 4 的玩家移动系统。

要求：
1. 创建 Player.tscn 和 player.gd。
2. 使用 CharacterBody2D。
3. 实现 2D 俯视角移动。
4. 支持 WASD 和方向键。
5. 添加 Camera2D 跟随。
6. 代码保持简单、适合初学者理解。
7. 不要实现背包、战斗、采集、NPC。
8. 完成后告诉我：
   - 改了哪些文件
   - 脚本挂在哪个节点
   - 在 Godot 中如何测试

---

## 3. 第一张地图

请创建第一张测试地图 TeaHouseMap.tscn。

要求：
1. 这是一个 2D 俯视角地图。
2. 地图包含：
   - 茶铺主体区域
   - 门前小路
   - 一小片树林
   - 一处溪流或取水点
3. 暂时使用占位色块或简单节点，不依赖真实素材。
4. 加入 Player 实例。
5. 预留 TileMap 或 TileMapLayer 节点。
6. 加入基础边界碰撞。
7. 不要制作复杂玩法系统。
8. 完成后告诉我：
   - 改了哪些文件
   - 如何将该场景设为主场景
   - 下一步建议

---

## 4. 采集系统

请实现一个最小可用的采集系统。

要求：
1. 创建 Collectible.tscn 和 collectible.gd。
2. 玩家靠近可显示“按 E 采集”。
3. 按 E 后物品消失。
4. 支持三种资源：
   - tea_leaves
   - spring_water
   - firewood
5. 先用简单信号或方法把采集结果发送给背包系统。
6. 不做复杂动画。
7. 完成后告诉我：
   - 改了哪些文件
   - 如何在 Godot 中摆放采集物
   - 如何测试

---

## 5. 背包系统

请实现一个最小可用的 Inventory 系统。

要求：
1. 支持添加物品。
2. 支持记录数量。
3. 支持显示在 HUD 左上角。
4. 只显示物品名和数量。
5. 不做格子背包，不做拖拽，不做分页。
6. 与采集系统联动。
7. 完成后告诉我：
   - 改了哪些文件
   - 如何测试采集后数量变化
   - 后续如何扩展

---

## 6. 制作系统

请实现一个最小可用的制作系统。

要求：
1. 在茶铺中设置一个制作交互点。
2. 玩家靠近后按 E 打开简单制作面板或直接制作。
3. 先只支持一个配方：
   - tea_leaves + spring_water = simple_tea
4. 制作时正确扣除原材料并增加成品。
5. 界面尽量简单。
6. 不做复杂菜单系统。
7. 完成后告诉我：
   - 改了哪些文件
   - 如何测试配方逻辑
   - 还有哪些地方适合优化

---

## 7. 售卖系统

请实现一个最小可用的售卖系统。

要求：
1. 在茶铺柜台或旅人处设置售卖交互点。
2. 玩家靠近后按 E 可以卖出 simple_tea。
3. 卖出后背包减少，金币增加。
4. 在 HUD 中显示金币。
5. 第一版只支持卖出 simple_tea。
6. 不做复杂经济系统。
7. 完成后告诉我：
   - 改了哪些文件
   - 如何测试售卖
   - 后续可以如何扩展更多商品

---

## 8. 昼夜循环

请实现一个最小可用的昼夜系统。

要求：
1. 增加一个时间变量。
2. 随时间推进，画面颜色逐渐变化。
3. 建议使用 CanvasModulate 或类似简单方式。
4. 在床边设置睡觉交互。
5. 按 E 睡觉后进入下一天。
6. 保留背包和金币数据。
7. 不做复杂日历系统。
8. 完成后告诉我：
   - 改了哪些文件
   - 如何测试昼夜切换
   - 如何测试睡觉进入下一天

---

## 9. 氛围优化

请为当前 Godot 2D 场景添加轻量氛围效果。

要求：
1. 添加茶铺暖光。
2. 添加简单雾气或飘叶粒子。
3. 使用适合 2D 像素风的方式。
4. 不要引入复杂后处理。
5. 不要影响已有玩法。
6. 完成后告诉我：
   - 改了哪些文件
   - 如何在 Godot 中测试效果
   - 性能上有什么注意点

---

## 10. 让 Codex 检查项目

请阅读 AGENTS.md、README.md 和 docs/design.md，然后检查当前项目结构是否适合继续开发。

要求：
1. 不要写玩法代码。
2. 不要重构现有文件。
3. 只检查目录、文档和下一步计划。
4. 给出最适合下一步执行的一个最小任务。
